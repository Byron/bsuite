Known Limitations

 General
ByronsPolyTools are intended to be a fast and flexible modeling solution. Therefore, I kept the workload as low as possible.
Because for every split a new mesh has to be build, I decided to work only on vertices and faces.
This is why the normals and the UVs of the mesh will be discarded.
When the mesh input of a BPTNode changes, this change won't be reflected. This is because this may change topology, which could make the result of the node unpredictable.
Workaround: None

 Smart Split
If connecting vertices, only two vertices may share a face to be connected. When there are more vertices sharing an edge, only two of them will be connected
Workaround: None

If you only select two edges, and these edges belong to the same face, than the SmartSplit will poke faces (instead of connecting these two edges which might have been your intention). This is because the SmartSplit acts on faces, so your two edges will be converted to faces which results in mostly two single faces that are poked.
Workaround: Use the SplitPolygon Tool instead


 Solid Chamfer/Chamfer
In general, the beveling of edges which connect winged vertices (i.e. vertices with only two connected edges) may lead to unexpected results and should be avoided.
Workaround: None
When you try to chamfer boundary edges in combination with with edges that connect winged vertices (i.e. the corners of a plane), this will lead to quite a destroyed mesh.
Workaround: None

 Boundary
Boundary cannot select Edges that lie on a boundary.
Workaround: None

 Smart Move Component
You can not move winged vertex or border vertices or a single edge.
Workaround: None

 Soft Selections/ Falloff Modification
Masking: When the mask gets to big, then shrinking the mask may not restore the original selection
Workaround: Set maskSize to 0 and set then grow it again.
Masking: Grow and Shrink (on which masking is actually based on) may be slow on large meshes
Workaround: None
Information:
When you use the softselection and change your selection, then the falloff- and meshVisualisation will automatically be turned off. This will happen only once.
These parameters should always be turned off when you go on modeling. If falloff Visualization should stay on and you apply another softselection, then it will work on these vertex colors.
If you had have custom vertex colors, these would be lost then.
Workaround: Always turn off the Visualization parameters before going on with the modeling

 Soft Transformation Tool
If you are using the tool in the face component mask, then your selection may not correspond to the vertices that are moved then, but only if you use Maya 4.5 and whether there is a smooth node on top of the history stack. It seems to be a Maya internal bug, since it is fixed in Maya 5.
Workaround: Use edges or vertices instead.
