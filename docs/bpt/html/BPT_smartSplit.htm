SmartSplit

Split through every Selection, including vertices, edges and faces


ByronsPolyTools -Flag ...
LongName ShortName Description
-smartSplit -sp - split through your componentSelection
LongName ShortName --Additional Flags--
-relative -r - make the split and its slide in relative mode. This flag is should be turned on to prevent unexpected results
Example: ByronsPolyTools -er -r
-slide -sl - explicitly set the initial slideValue. If flag not set, it will default to 0.5
Example: ByronsPolyTools -er -sl 0.3
LongName ShortName --General Flags--
-slideNotLimited -snl - the slide parameter will work without its bounds (0 to 1)
-boostPerformance -bp - increases the general deformation performance by up to 20%

 Philosophy


With the smartSplit command I intended to create a tool which is able to split trough every selection in an efficient manner. It automatically checks for your selected component(s) and does a split which creates as many quads as possible. It will work with every mesh topology, no matter whether you split through quads, triangles or nGons.

After the split, you can procedurally change it by just changing values. Slide it, or lift it - to name just two possibilities.

The slide will automatically be adjusted to be clearly defined, which means that only those paths will slide, whose slide direction is clear. If paths are blocking each other, slide will be disabled for these sections, while others can still slide when appropriate.

The slide or slideAlongNormal can be relative or absolute, which can be set independently of each other. If you change slide to absolute mode, it will keep a specific distance to one of its boundary edges, allowing to create an "offset edge" relative to the boundary. When you change the slideAlongNormal into absolute mode, the normal offset of the vertices is dependent to the face(s) they originate from: bigger faces will have a bigger offset then small ones, while in relative mode, every vertex will have the same distance to the surface.

 Abilities for ...

Because of the split's "smart" feature, the command will adapt to your component selection, doing only appropriate actions. These are sometimes specific to components and/or the relations between these components. For that reason, it will be figured out per component.

 
 Vertex Selections
 
When vertices are selected, it checks whether there is the possibility to make a connection (create a new edge between vertices) between them. This is true when a pair of vertices does not share an edge.
If they do share an edge, the plug will find the face border of them and invoke the split.

 Examples

Vertices that don't share an edge with another selected vertex will be connected through an edge.
Note: There can be only one face per pair of vertices that can be split. This means, that one face may only be shared by to vertices to be split.



When a vertex shares an edge with another selected vertex, there will be a split through the faces around that interconnected cluster of vertices.



The plug will automatically determine which type of split will be appropriate.
Note:If a vertex could be part of a connection to another selected vertex and is part of a cluster, the cluster membership will determine the split.


 
 Edge Selections
 
In general, the edges will be converted to faces. The plug will check whether there are more contained then connected faces or vice versa and will use the dominant face Type as base for the split.
Note: When you select edges that imply both connected faces (due to an edge line for example) and contained faces (due to an edge Ring for example) it will use the dominant type of faces and discard the other one.

With single faces selected (faces that do not have selected neighbors), the split will actually poke the face. For quads and triangles, this will create a triangulated poke, but applied to nGons it will create as many quads as possible according to the face's vertex count.




 
 The Slide
 

When the split is done, there will be a new node in the stack of your mesh called ByronsPolyToolsNode. It holds the following attributes:

   • slideRelative
    • normalRelative
   • side
   • alongNormal
   • slide

With these attributes you will be able to quickly change the appearance of the split according to your needs.
There are also some parameters for the built in soft selections. Click here to learn more about it.
Please see the following for further information.

 Slide and Normal
The slide parameter will move the split into the direction of its boundary edges, while the alongNormal parameter will move them upwards or downwards.
Please Note: Whether a face may slide or not depends on how it is situated in the mesh. Sometimes, with a certain constellation of splitted faces, a slide direction may not be conform in comparison to the other faces. In that cases, the slide will be blocked out for the affected faces.

 Slide Relative

If slideRelative is zero, then the split edge will stick to one of their boundary edges. Its distance relative to it will always be the same, and can be tuned with the slide parameter.
When you set this value to one, then the slide will be relative to the underlying edge, i.e. 0.5 will be the center of this edge, no matter which size it has.

 NormalRelative

If normalRelative is zero, then the strength of the displacement along the normal is relative to the size of the underlying face. When turned to one, its displacement will be the same on all the faces, independently of their size.
