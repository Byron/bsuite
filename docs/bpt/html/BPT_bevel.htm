Bevel

Bevel the selected edges either leaving the original edge (solid Chamfer) or not (chamfer)


ByronsPolyTools -Flag ...
LongName ShortName Description
-solidChamfer -sc - chamfer edges keeping the original edges
-chamfer -c - chamfer edges deleting the original edges
LongName ShortName --Additional Flags--
-connectEnds -ce - Determines whether to connect the endings of the edge selection with the neighbour faces or not
Example: ByronsPolyTools -sc -ce
-triangulateEnds -te - Determines whether to triangulate the quad created by the "connectEnds" Flag or not
Example: ByronsPolyTools -sc -ce -te
-slide -sl - explicitly set the initial slideValue. If flag not set, it will default to 0.5
Example: ByronsPolyTools -c -ce -te -sl 0.3
LongName ShortName --General Flags--
-slideNotLimited -snl - the slide parameter will work without its bounds (0 to 1)
-boostPerformance -bp - increases the general deformation performance by up to 20%

 Philosophy

The bevel feature consists of two commands: Solid Chamfer and Chamfer. The first one will create a "mirrored" edge along each side of the original edges, and the second one will do nearly the same, except that it deletes the original edge.

As a general principle, chamfer and solid Chamfer will only change the topology of the selected edges, all the other ones will stay as they are (as opposed to the chamfer behavior of wings3d or the native Maya bevel {which is quite out of competition ;-) }).And it will connect the ends of the chamfer/solid chamfer with the next available vertex (leaving 3 quads by default and if possible), while wings3d or Maya bevel will create a new vertex, leaving a triangle and two nGons normally which would have to be fixed.

The Slide feature works with them, too, meaning that the slide attribute will change the " softness" of the edge, while alongNormal will move the original edge (in case of solid Chamfer) respective the two new edges (in case of chamfer) outwards or inwards. And of course, again your are free to select edges, faces or vertices and apply the command, because the selection will be implicitly converted to edges.

 Chamfer
 
The chamfer command will bevel the selected edges, deleting the original edge.
The slide parameter will control the offset relative to its original edge, and alongNormal will be mapped to all the new edges.
Important:
Before beveling you should make sure that your object is not non-uniformly scaled.
Because the chamfer makes it's calculations in objectSpace (the space without any transformation applied), this will lead to nonUniformly distorted bevels.
If this situation applies, please freeze the scaling and then redo the operation.
Tip:
In case the new edges vary in distance you can turn the slideRelative parameter to zero which will make all the vectors the same length and therefore correct this.

 Solid Chamfer
 
The solid chamfer command will bevel the selected edges, keeping the original edge.
The slide parameter will control the offset relative to its original edge, and alongNormal will be mapped to the original edge.

 The Slide
 

When the chamfer/solid chamfer is done, the ByronsPolyToolsNode on top of your mesh allows you to slide the created edge(s). The following attributes are useful for that:

   • normalRelative
   • alongNormal
   • slide

The slide Parameter will change the offset of the newly created edges, creating a softer or harder crease. With the alongNormal attribute you will be able to move the edges in- and outward.
