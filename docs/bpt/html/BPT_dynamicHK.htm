Dynamic Hotkeys - Additional Information

Understanding the procedure:

generateCmdString(string $command,int $type));

$Command will be a string for the main command, like "-sp" (smartSplit) or "-er" (edgeRing)

$type will be an int representing the submenu, or type of the supplied $command, i.e.
0 = smartSplit
1 = bevel
2 = selection
3 = transformation

The $type ID is necessary to be able to generate a "clean" command string, meaning that flags which are useless for a certain type of command will be ommited. This is for brevity only - the ByronsPolyToolsCmd won't consider these "useless" flags if they were given.

 Dynamic Command List

Just map one of these to a hotkey:

eval(generateCmdString("-sp ",0));	// = smart Split

//Bevel

invokeBevelCmd(generateCmdString("-c ",1)); // = chamfer, currently the bevel commands use an additional procedure

invokeBevelCmd(generateCmdString("-sc ",1)); // = solid chamfer



//Selection

eval(generateCmdString("-er ",2));	// = edgeRing

eval(generateCmdString("-el ",2));	// = edgeLoop

eval(generateCmdString("-bnd ",2));	// = boundary

eval(generateCmdString("-gro ",2));	// = grow

eval(generateCmdString("-shr ",2));	// = shrink



//Transformation

eval(generateCmdString("-smc ",3) );	// = smart Move Component

setSoftTool();				//   enter the soft transformation tool
